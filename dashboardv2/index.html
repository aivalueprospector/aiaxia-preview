Twin Management Dashboard

Prototype for issue #2. Plain utility admin styling, multi-page set. Brand toggle (bottom-right) swaps tokens; layout never reflows. Login & sign-up routes are blocked by design — see /login only.

Unauthenticated

  • LoginSign in to brand. No self-signup link.

Org Admin (on brand host — proasiste.com / eduasiste.org)

  • OverviewSeat counters, recent activity, latest twins.
  • TwinsProvisioned twins. Deprovision keeps user account.
  • Provision TwinCreate employee + provision twin in one step.
  • MembersOrg employees. Set passwords. Invite.
  • ProfileName, language, password. Organization name (org_admin only).

Business Partner (on admin.aivp.com)

  • OverviewOrgs managed + own twin.
  • OrganizationsList of orgs created. Quota usage. Brand chip.
  • Create OrganizationNew org + first org_admin + Stripe customer.
  • Organization DetailDrill into one org. Reset passwords. Provision twins.
  • ProfileSelf-service. No org-name edit.

Admin (on admin.aivp.com)

  • OverviewGlobal counters: orgs, twins, users.
  • All OrganizationsEvery org across brands. Search. Brand chips.
  • All UsersCreate any role. Edit any user. Reset password.
  • Business Partner ManagementCreate/edit FDSRs, reset password, assign clients.
  • Clients Upgrade TierView and change subscription tier per client.
  • Stripe Discount CodesCreate / list promotional codes via Stripe API.
  • Twin Billing SummaryAggregated seats × tiers × Stripe charges (read-only).
  • Twin ManagementGlobal twin inventory across all orgs.
  • Deleted ClientsSoft-deleted orgs with restore + purge actions.
  • ProfileAdmin self-service.

Non-admin org roles

  • Not a dashboard pageorg_lawyer, org_teacher, org_student, org_mentor, org_parent, org_child all log in and are redirected to their twin URL. They never reach the dashboard. Logic lives in CustomRedirectIfAuthenticated.

Notes

  • No billing page Billing is handled in Stripe directly. The dashboard never surfaces invoices or payment methods. organizations.seat_quota is written by the Stripe webhook (customer.subscription.updated); the dashboard only reads it.
  • Legacy pages removed Sidebar contains none of: Imagine, Map, Connect, Deliver, ProAssistant, ProGraph. These routes are deleted per issue #2 §6.
  • Brand swap Toggle (bottom-right) is prototype-only. In production, ResolveBrand middleware reads the host and sets data-brand on <html>. Layout, type, and density are identical across all three brands; only palette swaps.
